Теоретическая часть: Объектно-ориентированное программирование (ООП) — это парадигма программирования, основанная на концепции объектов, которые могут содержать данные в виде атрибутов (свойств) и методы для выполнения операций с этими данными. В ООП используется ряд принципов: Инкапсуляция — скрытие деталей реализации объекта и предоставление только необходимых интерфейсов для взаимодействия с ним. Наследование — возможность создания новых классов на основе существующих, что позволяет повторно использовать код. Полиморфизм — способность объектов разных классов обрабатывать одинаковые сообщения (методы) по-разному. Для данной работы основной акцент был сделан на создание классов и инкапсуляцию. Практическая часть:

Описание программы: В программе реализована система учёта студентов, которая позволяет управлять студентами и их оценками через два класса: Класс Student — представляет студента, его имя, группу и средний балл (GPA). Для этих атрибутов используется инкапсуляция, что ограничивает прямой доступ к ним. Вместо этого, доступны методы для работы с данными: обновление GPA и вывод информации о студенте. Класс Group — представляет учебную группу, содержащую список студентов. Он предоставляет методы для добавления и удаления студентов, а также для вывода всех студентов и поиска студентов с высоким GPA.
Реализация классов и методов: Класс Student: Атрибуты: имя, группа и GPA. Методы: init: инициализация атрибутов студента. display_info: вывод информации о студенте. update_gpa: изменение GPA студента, с проверкой на корректность значения. get_name, get_group, get_gpa: методы для получения значений атрибутов. Класс Group: Атрибут: список студентов. Методы: add_student: добавление студента в группу. remove_student: удаление студента из группы по имени. show_all: вывод всех студентов. get_top_students: возвращает список студентов с GPA выше заданного порога.
Пример работы программы: Программа создаёт несколько объектов Student, добавляет их в группу, выводит всех студентов, а затем фильтрует студентов с высоким GPA. Результаты сохраняются в файл students.json. Пример вывода: Список студентов: Студент: Володя, Группа: ИС-22, GPA: 3.6 Студент: Ерлан, Группа: ИС-23, GPA: 2.9 Студент: Камила, Группа: ИС-24, GPA: 3.95
